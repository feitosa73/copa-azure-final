## Story

**Como** aluno do workshop "Living Lab Azure-Native",
**eu quero** completar a Fase 6 em 8h implementando uma nova rota `/flow` no frontend Vite que mostra, em tempo real via SignalR, uma "bolinha" animada percorrendo **Gateway YARP → Function Entry → Service Bus → Function Consumer → n8n → SQL** para cada compra v2,
**para que** eu compreenda distributed tracing, correlation IDs, observabilidade com App Insights, e termine o workshop com a visualização mais didática possível do que aprendi nas 5 fases anteriores — capaz de demonstrar a um leigo a complexidade desacoplada que existe por trás de "comprar um ingresso".

## Dev Notes

### CORRECAO CRITICA: Gateway YARP e o No Zero (nao APIM)

O draft original desta story (v0.1) referenciava **APIM** como o primeiro nó do fluxo e como o componente que injeta `X-Correlation-ID`. Isso estava ERRADO. A arquitetura real (pós re-escopo ADE-004, confirmada na implementação de F2) é:

**APIM não existe no EPIC-002.** O nó zero do fluxo é o **Gateway YARP** (`src/Fifa2026.V2.Gateway/`), um projeto ASP.NET Core + `Yarp.ReverseProxy` hospedado em Container App por aluno.

O `X-Correlation-ID` é injetado pelo gateway em **`src/Fifa2026.V2.Gateway/Program.cs`** via `AddRequestTransform`:
```csharp
// Trecho real confirmado:
var incoming = transformContext.HttpContext.Request.Headers[CorrelationHeader].ToString();
var correlationId = string.IsNullOrWhiteSpace(incoming)
    ? Guid.NewGuid().ToString()
    : incoming;
transformContext.ProxyRequest.Headers.TryAddWithoutValidation(CorrelationHeader, correlationId);
// Devolve o mesmo correlationId ao cliente:
transformContext.HttpContext.Response.Headers[CorrelationHeader] = correlationId;
```

Portanto, o Flow Visualizer deve mostrar **6 nós** na seguinte ordem:

```
[0] Gateway YARP     ← NÓ ZERO: injeta X-Correlation-ID, valida JWT (F3)
[1] Function Entry   ← PurchaseEntryFunction: publica no Service Bus
[2] Service Bus      ← tickets-purchase: mensagem com CorrelationId em ApplicationProperties
[3] Function Consumer ← PurchaseConsumerFunction: grava no SQL, dispara n8n (F4)
[4] n8n              ← Workflow post-purchase-notification: log estruturado
[5] SQL              ← purchases.correlation_id gravado
```

### Event types para FlowEventsFunction

```
GATEWAY_YARP_RECEIVED    (timestamp, durationMs, status, httpMethod, path)
FUNCTION_ENTRY_PROCESSED (timestamp, durationMs, status, correlationId)
SERVICE_BUS_PUBLISHED    (timestamp, messageId, queueName)
FUNCTION_CONSUMER_DONE   (timestamp, durationMs, status, insertOutcome)
N8N_WEBHOOK_TRIGGERED    (timestamp, durationMs, workflowId, executionStatus)
SQL_INSERTED             (timestamp, purchaseId, entraOid)
```

### Correlation propagation matrix (estado final com todos os 6 componentes)

| Hop | Componente | Lê de | Escreve em | Confirmado em |
|---|---|---|---|---|
| Gateway YARP → Function Entry | header `X-Correlation-ID` | Gerado se ausente (novo GUID) | Request header downstream + Response header ao cliente | `src/Fifa2026.V2.Gateway/Program.cs` |
| Function Entry → Service Bus | header `X-Correlation-ID` | Request header recebido | `message.ApplicationProperties["CorrelationId"]` + `ILogger.BeginScope` | `src/Fifa2026.V2.Functions/Functions/PurchaseEntryFunction.cs` |
| Service Bus → Function Consumer | ApplicationProperties | `message.ApplicationProperties["CorrelationId"]` | `ILogger.BeginScope` + `purchases.correlation_id` | `src/Fifa2026.V2.Functions/Functions/PurchaseConsumerFunction.cs` |
| Function Consumer → SQL | BeginScope | `ILogger.BeginScope` scope | `purchases.correlation_id` | Schema `phase-01.sql` |
| Function Consumer → n8n | JSON body | BeginScope/mensagem | `payload.correlationId` no webhook | S2.4 AC-7 |
| n8n → log | JSON body | `payload.correlationId` | n8n execution log + Set node | n8n workflow |

### SignalR setup mínimo

- Service Mode: Default (não Serverless)
- Hub: `FlowHub` (classe em `src/Fifa2026.V2.FlowEvents/`)
- Método server→client: `SendFlowEvent(correlationId, eventType, timestamp, durationMs, payload)`
- Client subscribe: `connection.on("FlowEvent", handler)`
- Connection string: App Setting `AzureSignalRConnectionString`

### Troubleshooting conhecido

| Sintoma | Causa provável | Solução |
|---|---|---|
| Bolinha não aparece para um nó | Trace não chegou ao App Insights | Verificar se o componente tem App Insights configurado; aguardar latência de ingestão (até 2min) |
| SignalR não conecta | WebSocket bloqueado | Ativar fallback polling (2s); verificar CORS no SignalR Service |
| Nó "Gateway YARP" ausente no diagrama | Referência antiga ao APIM | Confirmar que o diagrama usa os 6 nós corretos (Gateway YARP, não APIM) |
| correlationId não aparece no App Insights | BeginScope não configurado em algum componente | Verificar ILogger.BeginScope com CorrelationId em todas as Functions |
| Latência > 30s para bolinha completar | App Insights ingestão lenta | Usar near-real-time query ou pré-popular eventos via polling |
